Git e GitHub

Após ter instalado o Git no seu computador, abra o prompt de comando e digite git --version para verificar a versão do Git instalada.

Se a instalação do Git estiver correta irá aparecer uma tela parecida com a imagem acima.

Configurações básicas

Digite o comando git config --global user.name 'nome do usuário'.

Após configurado seu nome, digite o comando git config --global user.name para mostrar seu nome de usuário.

Agora digite o comando git config --global user.email 'informe seu email'. Para verificar seu email digite git config --global user.email.

GitHub

Crie uma conta no GitHub e depois crie um repositório para salvar seu projeto na nuvem.

Após seu projeto ser criado no GitHub você deve vincular este no seu computador.

Crie uma pasta no seu computador para salvar seu projeto local.

Dentro da pasta que você criou, clique com botão direito do mouse e selecione Git Bash Here.

Agora na janela do terminal que abriu digite git init.

git init- este comando faz com que seu projeto seja monitorado pelo Git, a partir de agora qualquer alteração feita nesta pasta poderá ser restaurada a qualquer momento.

Agora digite git remote add origin'endereço do seu projeto' para vincular seu repositório GitHub à pasta local, que você criou para salvar seu projeto.

O comando acima está disponível no GitHub, após seu projeto ser criado.

Feito isto, você poderá fazer alterações local e enviar para o GitHub e acessar seu projeto de qualquer computador.

Commit

Quando é feita alguma alteração no projeto é usado o comando git commit 'mensagem' para identificar as alterações feitas, e caso algo dê errado, é possível voltar ao estado anterior. Portanto é recomendado que a cada alteração relevante seja feito um commit.

Push

Para enviar a versão atual do seu projeto para GitHub use o comando git push origin master

Resumo

Até agora aprendemos a fazer as Configurações básicas do Git, também aprendemos a iniciar o versionamento de um projeto, fazer commit e enviar nosso projeto para o GitHub

Pull

Para atualizar os arquivos locais do seu projeto usamos o comando git pull origin master. Este comando é útil quando temos outras pessoas colaborando com nosso projeto ou quando fazemos mudanças de forma remota no nosso projeto.

Diff

Este comando mostra a diferença entre os arquivos após alguma alteração

log

Este comando mostra os últimos commit e quem fez as alterações.Note que cada commit tem um hash identificação.

Git show

A identificação hash é utilizada para mostrar tudo o que foi feito naquele commit. Use o comando git show + hash.

Git reset

Se por algum motivo seja necessário desfazer alguma alteração no seu projeto, isso é possível utilizando o comando git reset + hash do commit que deseja descartar.
